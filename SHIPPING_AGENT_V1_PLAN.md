# 航运部 Agent v1 计划

## 1. 产品定位

航运部 Agent v1 的目标不是再做一个菜单式机器人，也不是让 AI 模拟点击网页，而是把现有航运业务系统升级为一个可通过飞书自然语言调用的业务 Agent。

核心定位：

> 飞书是自然语言工作入口，网页是数据复核、维护和导出入口，后端工具和数据层是 Agent 真正执行业务动作的底座。

用户理想体验：

1. 用户在飞书发送一句业务消息或上传文件。
2. Agent 理解用户意图。
3. Agent 调用后端白名单工具完成识别、查询、解析、生成草稿或报告。
4. 写入类动作先让用户确认。
5. 确认后数据沉淀到系统。
6. 网页端可以查询、复核、维护、导出。
7. 用户后续仍可在飞书继续追问。

## 2. v1 范围

v1 只聚焦三个高频闭环，先做稳，不扩散。

- [x] 商机货盘闭环：飞书一句话录商机，确认保存，网页可查，后续可问。
- [x] 新造船信息闭环：飞书一句话录新造船信息，确认保存，网页可查。
- [x] 标书闭环：飞书上传标书，Agent 解析，用户追问，确认保存成项目。
- [x] 查询闭环：Agent 能查询历史项目和市场情报，不凭空回答业务数据。
- [x] 管理闭环：网页端能复核 Agent 保存的数据，并能追踪来源。

暂不纳入 v1：

- [ ] 自动删除或覆盖已有记录。
- [ ] 复杂审批流。
- [ ] 多部门权限矩阵。
- [ ] 长期知识库/RAG。
- [ ] AI 自动操作浏览器页面。
- [ ] 非文本文件 OCR。

## 3. 总体架构

### 3.1 飞书入口

- [x] 接收飞书文本消息。
- [x] 接收飞书文件消息，v1 仅用于标书解析。
- [x] 识别用户确认、取消、修改、补充等连续对话动作。
- [x] 所有回复通过飞书发送给用户。
- [x] 普通聊天由配置好的 AI 自然回复，不返回固定菜单。

### 3.2 Agent 决策层

- [x] Agent 根据用户消息、文件、会话状态判断要调用的工具。
- [x] Agent 输出结构化 JSON 决策。
- [x] Agent 只能选择白名单工具。
- [x] Agent 不直接写库、不执行任意代码。
- [x] 如果不确定，Agent 追问用户，而不是瞎填。
- [x] AI 配置缺失或 AI 调用失败时，返回明确可操作错误。

### 3.3 后端工具层

v1 工具清单：

- [x] `chat_general`：普通自然语言回复。
- [x] `extract_cargo_opportunity`：识别商机货盘草稿。
- [x] `extract_newbuilding_info`：识别新造船草稿。
- [x] `update_pending_record`：根据用户补充或修改更新当前草稿。
- [x] `save_pending_record`：用户确认后保存当前草稿。
- [x] `search_project`：查询历史项目。
- [x] `search_market_records`：查询商机/新造船市场情报。
- [x] `parse_bid_file`：解析飞书上传的标书文件。
- [x] `answer_bid_question`：基于最近一次标书解析结果回答追问。
- [x] `save_bid_project`：用户确认后将标书解析结果保存成项目。
- [x] `generate_market_report`：生成商机或新造船市场分析报告。
- [x] `cancel_pending`：取消当前草稿或待办。

### 3.4 数据层

- [x] 市场情报记录保存到 `data/market_skill`。
- [x] 标书项目保存到 `data/projects`。
- [x] 飞书会话保存到 `data/feishu/sessions`。
- [x] AI 和飞书配置保存到 `data/config.json`。
- [x] Docker 部署持久化 `/app/data`。

## 4. 核心业务闭环

### 4.1 商机货盘闭环

目标：

> 用户飞书发一条货盘信息，Agent 识别字段，生成草稿，用户确认后保存到市场情报台账，网页端可查。

任务：

- [x] 用户发送自然语言货盘信息时，Agent 自动识别为商机货盘。
- [x] 抽取货品、吨数、装港、卸港、装载期、货主、业务模块、货盘类型等字段。
- [x] 字段不足时自然追问。
- [x] 字段足够时生成草稿回复。
- [x] 用户回复“保存/确认”后写入市场情报。
- [x] 用户回复“取消”后清空当前草稿。
- [x] 用户回复“装港改成宁波”等修改指令时更新同一草稿。
- [x] 用户问“刚才那条保存了吗”时能基于会话回答。
- [x] 网页端市场情报列表能查到飞书保存的记录。

验收标准：

- [x] 飞书发送“3000 吨甲苯，张家港到东莞，月底装”能生成商机草稿。
- [x] 回复“保存”后网页端市场情报出现该记录。
- [x] 记录中保留原始飞书消息和来源信息。
- [x] 再问“刚才那条保存了吗”能返回记录 ID 或保存状态。

### 4.2 新造船信息闭环

目标：

> 用户飞书发新造船资讯，Agent 识别字段，确认后保存到新造船台账。

任务：

- [x] 自动识别新造船相关消息。
- [x] 抽取船名、船厂、船东、DWT、船型、阶段、预计交付等字段。
- [x] 字段不足时追问。
- [x] 字段足够时生成草稿。
- [x] 用户确认后保存。
- [x] 网页端可查可维护。

验收标准：

- [x] 飞书发送一条新造船资讯能生成草稿。
- [x] 回复“保存”后网页端新造船台账出现该记录。
- [x] 记录中保留原始飞书消息和来源信息。

### 4.3 标书闭环

目标：

> 用户飞书上传标书，Agent 解析并支持后续追问，用户确认后保存为项目。

任务：

- [x] 用户飞书上传 PDF/DOC/DOCX/XLS/XLSX 并说明解析标书时，Agent 调用标书解析工具。
- [x] 解析结果保存到会话 `last_result`。
- [x] Agent 返回项目名称、投标截止时间、保证金、资格要求、风险提示等摘要。
- [x] 用户继续问“截止时间是什么”“保证金多少”“风险有哪些”时，基于最近解析结果回答。
- [x] 用户确认保存时，将解析结果保存到项目台账。
- [x] 网页端历史项目能查到该项目。

验收标准：

- [x] 飞书上传标书后能收到结构化摘要。
- [x] 追问截止时间能基于该标书回答。
- [x] 确认保存后网页端历史台账出现该项目。

### 4.4 查询闭环

目标：

> Agent 查询系统已有数据，不凭空回答业务数据。

任务：

- [x] 支持查询历史项目。
- [x] 支持查询商机货盘。
- [x] 支持查询新造船信息。
- [x] 支持按关键词、航线、货品、时间、状态查询。
- [x] 查询无结果时提示用户换关键词或补充条件。

验收标准：

- [x] 用户问“最近张家港到东莞甲苯货盘有哪些”时，Agent 查询市场情报后回答。
- [x] 用户问“福海创以前有没有项目”时，Agent 查询历史项目后回答。
- [x] Agent 不编造不存在的系统记录。

## 5. 企业级能力

### 5.1 操作日志

目标：

> 每次 Agent 做了什么都可追溯。

任务：

- [x] 新增 Agent 操作日志存储目录。
- [x] 记录飞书用户 open_id、chat_id、message_id。
- [x] 记录用户原文。
- [x] 记录 AI 决策 JSON。
- [x] 记录工具调用和工具结果。
- [x] 记录是否保存数据以及保存记录 ID。
- [x] 记录失败原因。
- [x] 日志中隐藏 API Key、飞书密钥、服务器路径等敏感信息。

验收标准：

- [x] 每次飞书 Agent 对话都会生成一条日志。
- [x] 保存商机后，日志能追溯到原始消息和记录 ID。
- [x] 日志不泄露密钥。

### 5.2 数据来源

目标：

> 系统里的数据能看出从哪里来、谁确认、何时确认。

任务：

- [x] 市场情报记录新增来源字段。
- [x] 标书项目记录新增来源字段。
- [x] 保存原始飞书消息。
- [x] 保存飞书 open_id/chat_id/message_id。
- [x] 保存确认时间。
- [x] 网页端展示来源摘要。

验收标准：

- [x] 网页端能看出某条市场情报来自飞书或网页。
- [x] 飞书保存的记录可追溯到原始消息。

### 5.3 安全与确认

任务：

- [x] 保存新记录前必须确认。
- [x] 删除记录 v1 不允许由 Agent 执行。
- [x] 覆盖已有记录 v1 不允许由 Agent 执行。
- [x] 文件解析失败时给出明确原因。
- [x] AI 返回异常时给出明确原因。
- [x] 飞书权限继续使用允许用户/允许群配置。

验收标准：

- [x] 用户未确认时不会写入市场情报或项目台账。
- [x] 用户说“删除这条”时 Agent 不直接删除。
- [x] 未授权飞书用户无法调用 Agent。
